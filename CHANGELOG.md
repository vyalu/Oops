## [1.0.1] — 2026-06-05

### Исправлено
- Устранены ложные сетевые запросы (404) с именами контрагентов/организаций при рендере карточек и редактировании — добавлена защита, не дающая некорректным значениям попадать в адрес изображения.
- Стабилизированы формы (организации, контрагенты, категории, каналы): убраны ошибки `null` при открытии и закрытии.
- Мобильная вёрстка: исправлено переполнение карточек и таблиц, добавлено выпадающее меню-гамбургер, кнопки в модальных окнах больше не выходят за край.

### Добавлено
- Проверка обновлений с GitHub (без авто-применения): кнопка «Проверить обновления» в разделе «Система».

## [1.0.0] — 2026-05-23

Первый публичный релиз. Самохостящийся трекер подписок и платежей для организаций.

### Основные возможности

**Учёт**
- Несколько организаций (юр. лиц) с раздельным учётом расходов.
- Контрагенты с логотипами: загрузка файла, авто-favicon с сайта, поиск с созданием на лету.
- Сотрудники как ответственные за подписки.
- Три типа записей: «Подписка» (регулярная оплата в день месяца), «Счёт/Баланс» (пополняемый счёт), «Разовый платёж».
- Категории и способы оплаты с иконками (встроенные SVG).

**Балансы и автоматизация**
- Получение баланса из внешних API: SMS.ru, BILLmanager (FirstVDS / FirstDedic) и произвольные JSON-источники с гибким парсером (вложенные пути, индексы массивов, валюта в строке).
- Обновление балансов по расписанию (каждые 30 минут).
- Списание баланса в день списания для счетов без API.
- Алерт при низком балансе (раз в день, пока баланс недостаточен).

**Уведомления**
- Четыре канала: Webhook, E-mail (SMTP), Telegram, Bitrix24.
- Напоминания о платеже: настраиваемое «начать напоминать за N дней», далее ежедневно (включая просрочку), пока оплата не отмечена.
- Флаг «Продлевается автоматически»: только предупреждение заранее, без напоминаний о просрочке.
- Кнопка «Отметить оплату» закрывает текущий платёжный период.
- Журнал доставки уведомлений с историей попыток.
- Устойчивость к перезапускам: пропущенные напоминания догоняются.

**Интерфейс**
- Два режима отображения подписок: крупные карточки и компактный список с раскрытием.
- Дашборд: предстоящие платежи, низкий баланс, расходы по организациям.
- Тёмная и светлая темы.
- Роли: admin / manager / viewer.

**Данные и эксплуатация**
- Импорт/экспорт: полный ZIP-бэкап (БД + логотипы + JSON), импорт из бэкапа Wallos.
- Восстановление из бэкапа через интерфейс.
- Автоматический бэкап БД раз в неделю; бэкап БД перед каждым обновлением.
- Обновление через веб-интерфейс загрузкой .tar.gz (без SSH).
- Healthcheck и ротация логов в Docker.

**Безопасность**
- HTTPS «из коробки» через встроенный Caddy (Let's Encrypt или самоподписанный для локальной сети).
- Автогенерация секретного ключа сессий при первом запуске.
- Secure-куки за HTTPS.
- Требование сменить стандартный пароль admin при первом входе.

**Развёртывание**
- Один Docker-контейнер приложения + Caddy. docker compose up -d --build.
- Полностью автономно: фронтенд-библиотеки и стили хостятся локально, работает без доступа в интернет.
