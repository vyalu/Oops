# Changelog

Все значимые изменения проекта документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
проект придерживается [семантического версионирования](https://semver.org/lang/ru/).

## [1.2.0] — 2026-07-06

### Добавлено
- Полностью новый визуальный стиль: тёплая светлая и тёмная темы, шрифты IBM Plex, обновлённые карточки и панели.
- Выбор основного цвета интерфейса (8 акцентов) и переключение светлой/тёмной темы в разделе «Система → Оформление». Логотип и все акценты меняются вместе с выбором.
- Новый тип подписки «Баланс (Ежедневное списание)» — для сервисов с посуточным списанием; стоимость указывается за месяц, порог уведомления настраивается в днях.
- Дашборд переработан: кольцо «Доли по организациям», график динамики расходов (копит историю по месяцам), горизонтальные бары по организациям, KPI «К оплате за 30 дней».
- Контекстные подсказки (?) у полей формы подписки с понятными объяснениями.
- Экран загрузки с анимированным логотипом.
- В календаре оплаченные платежи месяца отмечаются зачёркиванием и галочкой.
- Документация: инструкции по смене порта и доступу по доменному имени.

### Изменено
- Типы подписок переименованы для ясности: «Баланс (Периодическое списание)» и «Баланс (Ежедневное списание)».
- Меню сгруппировано в выпадающие разделы «Справочники» и «Администрирование», шапка в одну строку.
- Все цвета переведены на единую систему тем — интерфейс единообразен на всех экранах.

### Исправлено
- Корректный расчёт дат для годовых, недельных и других непомесячных циклов (напоминания и «предстоящие платежи»).
- После отметки оплаты платёж корректно исчезает из предстоящих до следующего периода.
- Устранены ошибки отрисовки графиков (SVG) и мелькание темы при загрузке страницы.

## [1.1.0] — 2026-06-13

### Добавлено
- Вкладка «Календарь» — платежи по месяцам в виде календарной сетки с листанием и итогом за месяц.
- Оценка «Хватит до» для счетов/балансов — примерная дата, до которой хватит текущего баланса при текущей стоимости.
- Меню сгруппировано: основные вкладки (Сводка, Подписки, Календарь) и выпадающие группы «Справочники» и «Администрирование».

### Исправлено
- Окончательно устранены ложные запросы (404) с именами в адресах логотипов — добавлена защита на уровне загрузки изображений.
- Стабилизированы формы подписок: убраны ошибки `null` в выпадающих списках контрагента, категории и способа оплаты.

## [1.0.0] — 2026-05-23

Первый публичный релиз. Самохостящийся трекер подписок и платежей для организаций.

### Основные возможности

**Учёт**
- Несколько организаций (юр. лиц) с раздельным учётом расходов.
- Контрагенты с логотипами: загрузка файла, авто-favicon с сайта, поиск с созданием на лету.
- Сотрудники как ответственные за подписки.
- Три типа записей: «Подписка» (регулярная оплата в день месяца), «Счёт/Баланс» (пополняемый счёт), «Разовый платёж».
- Категории и способы оплаты с иконками (встроенные SVG).

**Балансы и автоматизация**
- Получение баланса из внешних API: SMS.ru, BILLmanager (FirstVDS / FirstDedic) и произвольные JSON-источники с гибким парсером (вложенные пути, индексы массивов, валюта в строке).
- Обновление балансов по расписанию (каждые 30 минут).
- Списание баланса в день списания для счетов без API.
- Алерт при низком балансе (раз в день, пока баланс недостаточен).

**Уведомления**
- Четыре канала: Webhook, E-mail (SMTP), Telegram, Bitrix24.
- Напоминания о платеже: настраиваемое «начать напоминать за N дней», далее ежедневно (включая просрочку), пока оплата не отмечена.
- Флаг «Продлевается автоматически»: только предупреждение заранее, без напоминаний о просрочке.
- Кнопка «Отметить оплату» закрывает текущий платёжный период.
- Журнал доставки уведомлений с историей попыток.
- Устойчивость к перезапускам: пропущенные напоминания догоняются.

**Интерфейс**
- Два режима отображения подписок: крупные карточки и компактный список с раскрытием.
- Дашборд: предстоящие платежи, низкий баланс, расходы по организациям.
- Тёмная и светлая темы.
- Роли: admin / manager / viewer.

**Данные и эксплуатация**
- Импорт/экспорт: полный ZIP-бэкап (БД + логотипы + JSON), импорт из бэкапа Wallos.
- Восстановление из бэкапа через интерфейс.
- Автоматический бэкап БД раз в неделю; бэкап БД перед каждым обновлением.
- Обновление через веб-интерфейс загрузкой .tar.gz (без SSH).
- Healthcheck и ротация логов в Docker.

**Безопасность**
- HTTPS «из коробки» через встроенный Caddy (Let's Encrypt или самоподписанный для локальной сети).
- Автогенерация секретного ключа сессий при первом запуске.
- Secure-куки за HTTPS.
- Требование сменить стандартный пароль admin при первом входе.

**Развёртывание**
- Один Docker-контейнер приложения + Caddy. docker compose up -d --build.
- Полностью автономно: фронтенд-библиотеки и стили хостятся локально, работает без доступа в интернет.
